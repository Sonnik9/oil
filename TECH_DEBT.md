## GOAL

Написать скринер, который:

1. Символы берем из symbols.py

2. По ним находим (пока не знаю из каких источников) лимиты на open interest

3. Парсим текущий open intetrst по конкретной монете и сравниваем его с пороговым по данной монете.

4. Если текущее значение превышает пороговое на oi_threshold_percent * пороговое, то такую монету заносим в кеш списка искомых монет. если по этой монете в течение некоего временного окна будет снова сигнал мы его не дергаем. По истечение временного порога монету удаляем из кеша и она становится снова чувствительной к сигнальной проверке.


## CRITICAL:

1. Парсер в текущем виде не выполняет свою задачу ибо берет значения не из того источника, либо неправильно их парсит (для ясности будем логировать сам словарь ответа)

2. Добавишь логер на всех уровнях логирования (c_log.py)

3. Если парсить текущий опен интерест будем при помощи веб сокет стрима то понятие check_interval в секундах -- не актуально. Разве что, если данные о лимитах опен интереста отдаются при помощи рест запросов то можем делать запросы к актуальным лимитам раз в check_interval секунд

4. Если будем запускать стрима, он должен быть устойчивым к сетевым падениям, иметь поддержку автопинга, так же быть продуманным относительно лимитов. Запуск вебсокета будем разбивать на батчи/чанки.
